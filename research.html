First short Analysis:

  1. The fundamental question is what makes a good player?
  2. For most given statistical measures given on the link its some sort of regressions where each statistic gets an asigned weight of positive and negative importace.
  3. These regressions are run over years and a lot of players on data.
  4. We are given on the NBA the offense and defense scores but not on the WNBA
  5. We can imply the relationship between this scores and use that regression in order to get the same scores for the WNBA
  6. I know this can lead to a big argument against this way of modeling:
    • Female basketball can be very different than male basketball and what makes a good player on one side might not make it on the other.
    • While this is true I think it is fair to say that a good NBA player would be a good WNBA player if there was a transfer so we will still be capturing good relationships of stats and scoring

I know that I've been given links for common statistical measures in NBA. I believe I don't have enough data (team context) to try to accurately reproduce this statistics. I can't rerun a regression on historical data to get the coefficients in order to score and rank. If the data is proper (I know it's a big if) the offense and defense scores have been calculated using proper statistical models calibrated with historical data. I try to make a simple model capture as much as possible the relationship of the given data and the scorings (Since there is some true underlying relationship that involves this numbers).

def added_statistics(df: pd.DataFrame) -> pd.DataFrame:
    """
    Getting typical NBA statistics:
    - Percentages
    - True shooting Pct
    - Possesion usage
    """
    df = df.copy()
    df["reb"] = df["oreb"] + df["dreb"]

    # Shooting %
    df["fg_pct"]  = np.where(df["fga"]  > 0, df["fgm"]  / df["fga"], 0.0)
    df["fg3_pct"] = np.where(df["fg3a"] > 0, df["fg3m"] / df["fg3a"], 0.0)
    df["ft_pct"]  = np.where(df["fta"]  > 0, df["ftm"]  / df["fta"], 0.0)

    # True shooting
    ts_denom = (df["fga"] + 0.44 * df["fta"])
    df["ts_pct"] = np.where(ts_denom > 0, df["points"] / (2.0 * ts_denom), 0.0)

    # Usage proxy and per-minute rates
    poss_used = df["fga"] + 0.44 * df["fta"] + df["tov"]
    df["usg_per_min"] = np.where(df["minutes"] > 0, poss_used / df["minutes"], 0.0)

    for col in ["ast","stl","blk","reb","tov","pf","points","plus_minus"]:
        per_min = f"{col.lower()}_per_min"
        df[per_min] = np.where(df["minutes"] > 0, df[col] / df["minutes"], 0.0)

    # Normalize plus/minus a bit (optional)
    df["pm_per100min"] = np.where(df["minutes"] > 0, 100.0 * df["plus_minus"] / df["minutes"], 0.0)
    return df

Improved modelling

  1. We have used the most naive linear regression on the previous ranking.
  2. There are some issues with this:
  • Multicollinearity is very high both by construction of the features and by the nature of the features (e.g fgm, fga,fg3m,fg3a)
  • We haven't accounted for outliers that might mess with our model (e.g Players that have played less than x ammount of minutes aren't properly statistically tractable)
  1. We can now proceed with feature selection and using regularized linear regression for the collinearity (Ridge)

#Define a function to visualize the final equation:
def to_raw_equation(pipe, feat_names, name):
    ridge = pipe.named_steps["ridge"]
    scaler = pipe.named_steps["standardscaler"]
    coef_std = ridge.coef_.astype(float)
    intercept_std = ridge.intercept_.astype(float)
    beta = coef_std / scaler.scale_.astype(float)
    intercept = intercept_std - np.sum(coef_std * scaler.mean_.astype(float) / scaler.scale_.astype(float))
    # build string
    parts = [f"{name} = {intercept:.6f}"]
    for b, f in zip(beta, feat_names):
        parts.append((" + " if b >= 0 else " - ") + f"{abs(b):.6f}{f}")
    return "".join(parts)
